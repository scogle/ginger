For ginger to work properly, each view has to start with 'front matter'. (If you've used Jekyll you already know how it goes.) Basically, each view needs to start with a section like:

---
title: Top 5 Current NYTimes Best Sellers
id: best-sellers
description: The hardcover fiction list from the New York Times.
data_source: "/_data/nytimes_best_sellers.json"
---

(The --- are required before and after after your front matter)

After that, add your template to display your data. Take at look at this file in _views/02_best-sellers.html to see it in action.

Ginger views use a template language called Handlebars to display data. As a handy utility, we've also included Swag by default. Swag is a collection of super useful template helpers to use in constructing views. Yeeeahhh!

Once you've finished editing the view, run ginger build to generate your new site! Whoa! Or run ginger serve to build it and serve it locally! Even cooler!

Note - If you're using local data sources (ie, files stored in the _data directory) you will probably need to use ginger serve to prevent cross-site-origin-policy errors in your browser.
